YOKUWAKA 28: July, 2004

2004/07/01

　目黒駅西口の揚鳥屋でランチのセットを頼み、ドリンクをアイスコーヒーにした。店員がドリンクを渡す際になにか言ったのだが、よく聞きとれない。聞き返しても、同じように早口かつ小さい声で繰り返すだけ。3回目にして「シロップは、ひとつでよろしいですか?」と言っているのだと、ようやく理解する。質問の中身そのものもどうかと思うが(シロップは、ふつ～ひとつでええやろが)、聞き直しているのに同じように繰り返すだけという会話の仕方は一体どうしたものなのか? いいとか悪いとかいう以前に、理解不能の域に半分入り込んでいる気がしないでもない。

　ぢつわ揚鳥屋では、以前にも和光市駅前の店舗でも同様のことがあった。シロップのことではなかったけどね。確か、セットの組み合わせのことだったように思う。マックやロッテリア、モスバーガーやサブウェイでは同様の経験がないことを考えると、これは店員の個人的な質の問題ではなく、揚鳥屋自身のファーストフード接客方法そのものの問題ではないのか? 同様の現象が発生する可能性としては、上記のモスバーガーやサブウェイの方が圧倒的に高いと思うのだが、少なくとも自分自身の経験では「否」である。

2004/07/10

　RIMNETの方にMovable Typeを導入してみようかと、ファイルを準備。テストなので、バージョンは3.0β。ところが、FTPがうまくいかない。大量にファイルをputしようとすると、途中で止まってしまう(反応が、なくなってしまう)。この現象はいまにはじまったものではないので、すでに半分あきらめている。多分、こちら側のネットワークの問題。それよりも、ディスクの使用容量が6MB弱になっていたことの方がビックリ。そんなに使っていたとわね。全部で10MBなので、空き容量は4MB。Movable Typeのモジュール、入りきらないんぢゃないのか。

　仕方ないので、優先度の低いファイル(画像ファイルとか、アーカイブファイルとか)をいくつか削除して空きスペースをつくる。んでもって、Movable Typeのファイルを再度put(正確には、mput)。mt-load.cgiはうまく動いたのだが、mt.cgiを呼ぶとInternal Server Errorになる。なんでやねん。文字コードを変えたり、mt.cfgのPublishCharsetを変えたりしてみたがNG。ディスクスペースの容量制限ギリギリにもなってしまったし、このまま挫折する可能性が高い。どっか、別んとこで試してみるか……って、そんな場所どこにあんねん。

2004/07/11

　その後、imagesディレクトリ以外のファイルをすべてEUCでputし直したのだが、Internal Serverv Errorが出たり出なかったり……出ないときは、もちろん正常動作なのでブログの作成から記事の投稿まで可能。んが、とにかくマトモに動いてくれないことには話にならない。とゆーわけで、Internal Server Errorはなくならないしディスク容量もないっつ～ことで、Movable Typeは削除。エクスポート機能も、たいしたことないのでいいです。livedoorの方は「livedoor Blog PRO」だとエクスポート機能が使えるらしいんだけど、カネを払ってまで欲しい機能かというと否。HTML保存でいいかな、という感じ。
